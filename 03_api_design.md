## 1. API設計方針

本システムでは、SpringBootによりRESTfulなWebAPIを提供する。
クライアントは来店客向け注文画面、スタッフ向け注文管理画面、店長向け売上管理画面、本部管理者向け店舗管理画面の4種類があり、それぞれのクライアントから利用されるAPIを提供する。

API は JSON 形式でリクエスト・レスポンスを行う。
スタッフ・店長・本部管理者向けのAPIに関しては、Keycloak による認証済みの JWT を必要とする。

来店客向けAPIは Keycloak認証の対象外として、QRコード、来店セッショントークン、テーブルステータスにより注文可否を制御する。

## 2. 共通仕様

### 2.1 ベースURL

本システムのAPIは、以下のベースURLの配下に定義する。

| 環境         | ベースURL                       |
| ------------ | ------------------------------- |
| ローカル環境 | `http://localhost:8080/api/v1`  |
| AWS環境      | `https://{alb-dns-name}/api/v1` |

APIのバージョン管理を行いやすくするため、URLに `/v1` を含める。

本ポートフォリオでは Route53 による独自ドメイン取得は対象外とし、AWS環境では ALB のDNS名を利用してAPIにアクセスする。
HTTPS化、独自ドメイン利用、CloudFront による静的ファイル配信は将来的な拡張対象とする。

来店客向けAPI、店舗業務API、本部管理API、バッチ管理APIは以下のパスで分類する。

| 区分          | パス                | 説明                                                                          |
| ------------- | ------------------- | ----------------------------------------------------------------------------- |
| 来店客向けAPI | `/customer`         | QR注文画面、メニュー取得、注文登録など、来店客が利用するAPI                   |
| 店舗業務API   | `/stores/{storeId}` | テーブル管理、注文確認、会計、店舗別メニュー・売上管理など、店舗単位で扱うAPI |
| 本部管理API   | `/admin`            | 店舗管理、スタッフ管理、全店舗売上参照など、本部管理者向けAPI                 |
| バッチ管理API | `/batches`          | バッチ実行、バッチ実行履歴確認などのAPI                                       |

### 2.2 認証方式

認証方式は 店舗業務API、本部管理APIは Keycloak による JWT 認証を採用する。
来店客向けAPIは Keycloak 認証の対象外とし、QRコード、来店セッショントークン、テーブルステータスにより注文可否を制御する。

### 2.3 権限制御方針

権限制御は、ユーザーの役割に応じてアクセス可能なAPIを制御する。
店舗業務API、本部管理APIでは、JWTに含まれるユーザー情報を基に権限を判定する。
来店客向けAPIでは、QRコード、来店セッショントークン、テーブルステータスによりアクセス制御を行う。

店舗業務向けAPIでは、JWTに含まれるユーザー情報を基に、店舗スタッフと店長の権限を判定する。店舗スタッフは自店舗のデータのみ操作可能とし、店長は自店舗の売上やメニュー管理も追加で可能である。

また、本部管理者向けAPIでは、全店舗のデータにアクセス可能であり、店舗管理やスタッフ管理も行える。

そして、本部管理者は、店長、店舗スタッフの権限も内包しており、店長は店舗スタッフの権限を内包している。

それぞれのAPIの詳細においては後述の「3. API一覧」「4. API詳細」にて、各APIの権限制御について明記する。

### 2.4 リクエスト形式

リクエストは JSON 形式で送信する。
必要に応じて、HTTPヘッダーに認証情報やコンテンツタイプを指定する。

### 2.5 レスポンス形式

レスポンスは JSON 形式で返却する。
成功時はデータを含むオブジェクトを返し、エラー時はエラー情報を含むオブジェクトを返す。

### 2.6 エラーレスポンス形式

エラー発生時は、以下の形式でエラー情報を返す。

```json
{
  "error": "エラーコード",
  "message": "エラーメッセージ"
}
```

### 2.7 HTTPステータス方針

| HTTPステータス            | 用途                                 |
| ------------------------- | ------------------------------------ |
| 200 OK                    | 取得・更新成功                       |
| 201 Created               | 登録成功                             |
| 204 No Content            | レスポンスボディなしの更新・削除成功 |
| 400 Bad Request           | 入力値不正                           |
| 401 Unauthorized          | 未認証                               |
| 403 Forbidden             | 権限不足・自店舗外データへのアクセス |
| 404 Not Found             | 対象データなし                       |
| 409 Conflict              | 楽観ロックエラー、状態不整合         |
| 500 Internal Server Error | 想定外エラー                         |

## 3. API一覧

### 3.1 来店客向けAPI

| No    | 機能             | HTTPメソッド | エンドポイント                                                                     | 認証 | 必要ロール | 店舗制御                  | 概要                                                                               |
| ----- | ---------------- | ------------ | ---------------------------------------------------------------------------------- | ---- | ---------- | ------------------------- | ---------------------------------------------------------------------------------- |
| C-001 | QR注文画面表示   | GET          | `/customer/tables/{qrToken}`                                                       | 不要 | なし       | QRトークンで制御          | QRトークンから対象テーブルを特定し、注文画面表示に必要な情報を取得する             |
| C-002 | メニュー一覧取得 | GET          | `/customer/stores/{storeId}/menus?categoryId={categoryId}&page={page}&size={size}` | 不要 | なし       | storeIdで制御             | 来店客向けに、対象店舗の販売中メニュー一覧をカテゴリ・ページング条件付きで取得する |
| C-004 | メニュー詳細取得 | GET          | `/customer/stores/{storeId}/menus/{menuId}`                                        | 不要 | なし       | storeIdで制御             | 来店客向けに、対象店舗の指定メニューの詳細情報を取得する                           |
| C-005 | 注文登録         | POST         | `/customer/orders`                                                                 | 不要 | なし       | qrToken・visitTokenで制御 | qrToken と visitToken を検証し、有効な来店セッションに対して注文を登録する         |
| C-006 | 注文確認         | GET          | `/customer/orders/{visitToken}`                                                    | 不要 | なし       | visitTokenで制御          | visitToken を検証し、有効な来店セッションの注文内容を取得する                      |

| S-005 | テーブル別注文一覧取得    | GET          | `/stores/{storeId}/tables/{tableId}/orders?status={status}&page={page}&size={size}` | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定テーブルの注文一覧をステータス・ページング条件付きで取得する                                                      |
| S-006 | 自店舗メニュー一覧取得    | GET          | `/stores/{storeId}/menus?categoryId={categoryId}&page={page}&size={size}`           | 必須 | STAFF      | storeIdで制御 | 対象店舗の販売中メニュー一覧をカテゴリ・ページング条件付きで取得する                                                            |
| S-007 | 自店舗メニュー詳細取得    | GET          | `/stores/{storeId}/menus/{menuId}`                                                  | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定メニューの詳細情報を取得する                                                                                      |
| S-008 | 自店舗メニュー販売ステータス変更 | PATCH     | `/stores/{storeId}/menus/{menuId}`                                                  | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定メニューの販売ステータスを変更する                                                                               |
| S-009 | 注文一覧取得              | GET          | `/stores/{storeId}/orders?status={status}&page={page}&size={size}`                  | 必須 | STAFF      | storeIdで制御 | 対象店舗の注文一覧をステータス・ページング条件付きで取得する                                                                    |
| S-010 | 注文詳細取得              | GET          | `/stores/{storeId}/orders/{orderId}`                                                | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定注文の詳細情報を取得する。 テーブル別注文一覧画面から遷移する注文詳細も同じくこのAPIをたたく                      |

| S-012 | 注文更新                  | PUT          | `/stores/{storeId}/orders/{orderId}`                                                | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定注文を更新する。 注文ステータス変更、注文明細の追加・削除・更新などを行う                                         |
| S-013 | 注文ステータス変更API     | PATCH        | `/stores/{storeId}/orders/{orderId}`                                                | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定注文のステータスを変更する。 注文ステータス変更のみを行う場合に利用する                                           |
| S-014 | 注文削除                  | DELETE       | `/stores/{storeId}/orders/{orderId}`                                                | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定注文を削除する。 注文ステータスが ORDERED の場合のみ削除可能                                                      |
| S-015 | 会計金額確認              | GET          | `/stores/{storeId}/visit-sessions/{visitSessionId}/payment-amount`                  | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定テーブルの会計金額を確認する。 注文ステータスが ORDERED の注文の合計金額を返す                                    |
| S-016 | 会計処理                  | POST         | `/stores/{storeId}/visit-sessions/{visitSessionId}/payment-completion`              | 必須 | STAFF      | storeIdで制御 | 対象店舗の指定テーブルの会計処理を行う。 注文ステータスが ORDERED の注文を会計処理し、注文ステータスを COMPLETED に変更する     |
